CONFIGURATION

Application

  • The VI App operates alongside IBMâ€™s Maximo / MAS products and is fully integrated.
  • The VI App can be hosted on a separate cloud, WAN or LAN to the Maximo / MAS applications, but it is recommended that the VI App is installed on the same LAN as Maximo if possible.
  • Most of the configuration is done via the installation Manifest file. This excludes the configuration of the IoTservice which is performed separately using a DeviceMap container.
  • The installation process requires the capture of pre-requisite data. Namely
    1. Client Name and Address & PostCode
    2. Site(s) name, address, postcode, VI siteId
    3. Maximo OrgId & siteId
    4. 3D Viewer siteId
    5. assetType Attribute in Maximo
    6. Fully qualified URL of Maximo / MAS
    7. Fully qualified URL of the 3D Viewer
    8. The name of the database. This is defaulted to vilApp unless otherwise stated.
  • Execution of the docker installation scripts require permission to the VI Repo in GIT. VI Support will assist in this
  • Once the application installation scripts have been executed and completed, user log-on services will be enabled to the VI portal.
  • The first step is to execute the location map and asset type map in the Application Manager.

IoT

The IoT configuration

  1. Gateway ID
  2. Protocol
  3. MQTT username and password (VI provide this if using the VI MQTT Broker)
  4. Subscription ID
  5. MQTT Topic

DeviceMap

  1. The deviceId (or deveui) of each IoT device
  2. The appKey (LoraWan)
  3. The joineui or appeui (LoraWan)
  4. The device Name
  5. The device Location (see location hierarchy)
  6. deviceType (see IoT DeviceTypes)
  7. The path to the devicetype codec (lorawan only)
  • The IoT device types are maintained in the Application Manager where each datapoint is defined for the device type and which datapoints are static, dynamic and recordable.
  • The IoT Machine Learning service will be running in background. When the device Map is populated, the IoTDeviceMapper will build the IoT device architecture, listen to the gateway on the defined topic and start to record data from the devices.
  • Any error in the configuration will written to a log file for review by approved technicians.
  • The VI InfoHub widgets will have access to the devices and all recordable data points.
  • Are Maximo Meters used and deployed?
  1. See VI support on how to map devices and data points to the Maximo meters.
